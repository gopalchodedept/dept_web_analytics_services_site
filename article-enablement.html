Zero To Adobe
Certified.

DEPT runs a role-based, hands-on Adobe Enablement Program: three separate tracks, live internal sessions from our Adobe expert, a real sandbox with answer keys, and a certification target for every track. This entry explains the whole thing: how you get assigned, what you'll actually do week to week, and how long it honestly takes.

Entry 002~12 min readTraining · AEP · CJA · RTCDP + AJODEPT · Adobe Partner Enablement · v2 · July 2026

TL;DR

Three tracks, one per role: AEP for the people who build the platform, CJA for the analysts who report on it, RTCDP + AJO for the people who activate it. Tracks are assigned based on experience and interest; nobody does all three in sequence.

Every track has the same rhythm: live session → hands-on task in the DEPT sandbox → review → next session, ending in an Adobe certification.

Honest budgets: roughly 80 hours for CJA, 120 for AEP, 120 for RTCDP + AJO. Most of those hours are deliberate hands-on practice, not videos.

01What The Program Is

Because DEPT is an Adobe partner, we can package four things that usually live apart into one program. Live internal training: short webinar-style sessions, about 45 minutes each, run by DEPT's Adobe expert; concepts and a live walkthrough first, then a hands-on task to complete before the next session. Every session is recorded, so the recordings become an internal on-demand library for every future hire. Official Adobe courses: the self-paced on-demand videos on each certification path, which slot around project work. Hands-on practice: every trainee gets a DEPT sandbox with a frozen sample dataset, so labs have known-correct answers to check against. And certification prep: each track carries a specific exam target, prep courses, and a practice test.

02Which Track Will I Be On?

Tracks will be assigned, based on the experience and interest (onboarding steps 2-3), not all three in sequence. Use this table to see where each role lands. Anyone working near the platform gets the AEP concepts they need inside their own track.

03How Onboarding Works

The standardized workflow for every new trainee, from identification to certification. Same seven steps, every time:

  1. Identify the trainee and their upcoming project needs.
  2. Assess experience and interest: a short baseline conversation or questionnaire.
  3. Assign a track using the role table above.
  4. Provision sandbox access: the DEPT engineering lead requests AEP + CJA sandbox access.
  5. Schedule the internal live sessions: DEPT's Adobe expert runs them and assigns the hands-on tasks.
  6. Enroll in the track's Adobe on-demand courses (the DEPT engineering lead manages enrollment) alongside the sessions.
  7. Certify: practice test, then the live exam for the track's target credential.

04The Rhythm Every Track Follows

All three tracks run the same loop, repeated each session, and finish with a certification push. It's deliberately simple so it scales to any number of trainees and any platform: the expert explains and demos live using genericized project docs, you build it yourself in the sandbox between sessions and check your work against the frozen-dataset anchors, the next session opens by reviewing that work and building on it, and once the modules are done you move to the Adobe courses, the practice test, and the live exam.

05AEP By Role: Build It vs Read It

Everything in the Experience Cloud stands on Adobe Experience Platform, but the tracks need it at different depths, and this distinction is what keeps the program fast:

AEP track

Build It

  • Schemas, field groups, datasets, Tags, ingestion
  • For the people who set the platform up
  • The deepest and most technical treatment
CJA and RTCDP + AJO tracks

Read It

  • Opens with a one-session "AEP for readers" module
  • Read a schema, search for a field, copy a field path (for example, to create a new dimension in a data view), preview a dataset
  • No schema building, ever. Proven: it's exactly how our first analyst cohort ramped in one session

The foundation under all three: XDM schemas · datasets · identity & ECID · Real-Time Customer Profile · data governance.

06Track 1 · AEP

For data engineers & architects · the most technical track

The people who get data into the platform correctly. Two roles share this track with different emphasis: the architect designs the XDM schemas and the end-to-end flow (how data arrives, is stored, and is activated); the data engineer implements to that spec: the data layer push, Tags configuration, and source connections. Required background: comfort with web and data concepts and reading JSON. No prior Adobe experience needed.

What the sessions cover:

  • The Experience Cloud stack and where AEP sits
  • XDM: classes, field groups, data types; schema design by the architect, implementation by the engineer
  • Schemas and datasets: design and preview
  • Adobe Tags (Launch): property setup, data elements, rules, the Web SDK extension, publishing flow
  • Data layer to schema mapping: how Tags picks events off the data layer and maps them to XDM paths; when data skips Tags entirely (app code or mobile sends direct)
  • Web SDK and datastreams: how events reach the platform
  • Sources configuration: FTP, cloud storage, database and streaming (HTTP API / server-to-server) connections; gathering keys, endpoints and paths
  • Identity, ECID and the Real-Time Customer Profile
  • Ingestion: streaming vs batch (for example, AI-chat events often arrive as a batch push from the application team, not via the data layer)
  • Data governance basics

Representative hands-on tasks: build a schema with standard and custom field groups and create a dataset; configure a Tags property with one data element and one rule, mapping a data-layer value to an XDM path; send events (Web SDK or Postman) and confirm they land in dataset preview; trace an event's identity end to end.

Adobe courses: "Get started with AEP for data architects and data engineers" (Experience League, ~16 modules, about 5 hours). Tags has no standalone certification; it's covered inside the AEP foundations content and this track's live sessions.

07Track 2 · CJA

For analysts · the most in-demand track

Analysts who build reports and dashboards and answer business questions with data. You are an analyst? This is your track. It opens with the built-in AEP for readers module: read a schema, search for a field, copy a field path, preview a dataset; one session, no schema building. A background in Adobe Analytics or GA is helpful but not required.

What the sessions cover:

  • AEP for readers: schemas, datasets and identity, from the reading side
  • The mental-model shifts coming from Adobe Analytics or GA into CJA
  • Connections vs data views; the two-data-view model; sessionization
  • Derived fields and counting patterns (no native Occurrences)
  • Filters (segments) and calculated metrics
  • Workspace: panels, tables, visualizations, date ranges
  • QA methodology and dev-to-prod governance

Representative hands-on tasks: find 5 schema fields and their real values in dataset preview; create a data view and build derived fields, calculated metrics and filters; assemble a dashboard and validate every tile against the anchors; a capstone where you rebuild a mini dashboard solo and self-check.

Adobe courses: CJA Foundations (course 1309), CJA B2B Edition Foundations (course 1337), CJA Business Practitioner Professional (course 1062, CJA-BP200), and the practice test (course 1010).

08Track 3 · RTCDP + AJO

For activation & marketing practitioners

Practitioners focused on activation: building audiences, sending them to destinations, and orchestrating journeys and messages. It opens with the same one-session AEP for readers module as the CJA track, plus the Real-Time Customer Profile model. CJA familiarity helps for measuring what the journeys do.

What the sessions cover:

  • RTCDP: audiences and segmentation, merge policies

09Certification Tiers

Adobe uses a consistent two-tier model for each product: Business Practitioner Professional (entry level, about 6 months of experience) then Developer Expert (1 to 3 years, leaning heavily on AEP foundations). Every track above carries its own certification map; this program targets the Professional tier first, with Expert as a later stretch.

Before booking any exam

Exam IDs and course codes come from Adobe's course catalog. Confirm exact exam metadata (question count, duration, passing score, price) and any exam-ID revisions on a logged-in Adobe certification page before booking: Adobe periodically re-releases exams under new IDs. Example: the RTCDP Professional exam appears as both AD0-E602 and the newer AD0-E614.

10How Long It Takes

Honest planning numbers, as rough averages: ~80 hours for CJA, ~120 for AEP, ~120 for RTCDP + AJO to get trained and Professional-cert-ready. Converted to calendar time: one week at 100% availability is 40 hours, 50% is about 20 hours a week, 25% is about 10.

RTCDP + AJO (two Professional exams)~120 hours~12 weeks~6 weeks~3 weeks

Why these numbers are bigger than the certification math. The Adobe courses and live sessions account for roughly a quarter of each budget. The rest is deliberate hands-on sandbox practice, and that's on purpose: passing an exam and being safe on a client project are two different bars. The extra hours buy build-break-rebuild reps in the sandbox, the capstone builds, working through genericized real project docs, and review time with the expert. These are rough averages for someone with a general analytics background; calibrate per person after the first week.

What the hours cover, per track. CJA (~80h): roughly 6h of live sessions, 8h of Adobe courses, 55h of hands-on data-view, component and dashboard reps, and 10h of practice test and review. AEP (~120h): roughly 10h of sessions, 10h of courses, 85h of hands-on across schemas, Tags (Launch), data-layer mapping, sources configuration and identity tracing (the deepest lab work of the three), and 15h of review. RTCDP + AJO (~120h): two Professional exams, so roughly 10h of sessions, 12h of courses, 80h of hands-on across audiences, destinations, journeys, messages and decisioning, and 18h of prep spread over two practice tests.

11Adobe Partnership: Courses & Vouchers

As an Adobe partner, DEPT can route trainees to official learning and certification rather than piecing it together. The Adobe Partner Hub gives partners the on-demand training catalog and team enrollments for the courses on each certification path. Experience League offers free self-serve courses and per-product certification prep guides; the basic prep courses need no special credentials. Adobe Premium Learning is a paid tier bundling deeper content and certification vouchers.
